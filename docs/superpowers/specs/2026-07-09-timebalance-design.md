# TimeBalance — Design Spec

## Goal

Минималистичный оффлайн-трекер рабочего времени с автоматическим расчётом дневного и недельного баланса относительно нормы 525 мин (Пн–Пт).

## Confirmed Business Rules

| Rule | Value |
|------|-------|
| Дневная норма | 525 мин, Пн–Пт |
| Выходные | Сб–Вс, норма = 0 |
| Баланс дня | `отработано − норма` |
| Недельный баланс | сумма балансов дней текущей недели + перенос с прошлых недель **внутри периода** |
| Периоды | 8–22 и 23–7 числа; в расчёт входят только дни текущего периода |
| Сброс | 8-го и 23-го баланс обнуляется (история сохраняется, но не участвует в расчёте) |
| Уведомления | 6-го и 21-го: push в 11:00 + баннер при первом открытии |
| Прогноз ухода | `приход + (525 − баланс_периода_до_сегодня)` |
| Тема | тёмная по умолчанию |

## Assumptions (deferred to implementation review)

- Неделя начинается в понедельник
- Ручной ввод ЧЧММ — на главном экране и при редактировании
- Редактирование прошлых дней — по нажатию на день в списке
- Повторное нажатие «Я пришёл»/«Я ушёл» перезаписывает время
- Пакет остаётся `com.example.shikagera1` (переименование — позже)

## Architecture (recommended)

**Room + ViewModel + Compose**, чистая domain-логика без Android-зависимостей.

```
UI (Compose) → ViewModel → Repository → Room DAO
                    ↓
              Domain calculators (pure Kotlin)
```

### Alternatives considered

1. **DataStore only** — проще, но неудобно для выборки/редактирования истории дней. Отклонено.
2. **Room + domain layer** — рекомендовано: запросы по датам, тестируемая логика.
3. **SQLDelight** — избыточно для одной таблицы.

## Data Model

```kotlin
DayEntry(date: LocalDate, arrivalMinutes: Int?, departureMinutes: Int?, note: String)
```

Settings (DataStore): `lastWarningBannerDate` — чтобы баннер показывался один раз в день.

## UI — Main Screen

- День недели + дата
- Крупный недельный баланс (зелёный/красный)
- Кнопки «Я пришёл» / «Я ушёл»
- Поля ручного ввода ЧЧММ
- Прогноз ухода (после фиксации прихода)
- Список дней текущей недели (баланс + пометка)
- Баннер-предупреждение 6-го/21-го

## Notifications

- `WorkManager` + `NotificationManager`, канал «Напоминания»
- Разрешение `POST_NOTIFICATIONS` (API 33+)
- Текст: «Через 2 дня баланс будет сброшен (8-е / 23-е число)»

## Testing

Unit-тесты для: периодов, баланса, прогноза ухода, парсинга ЧЧММ.
